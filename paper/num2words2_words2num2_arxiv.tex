\begin{abstract}
`num2words2` and `words2num2` turn the old one-way `num2words` project into a
bidirectional text-normalization stack. The forward fork keeps the familiar API
while expanding the documented surface to more than 120 languages and 170+
locale keys, including aviation/ICAO profiles, cheque formatting, fractions,
string input handling, and per-call grammatical controls. The inverse fork
parses spoken-form numbers back into numeric values, rewrites numbers inside
sentences, and auto-parses mixed text that contains currencies, units, and
locale-dependent separators. This paper states the concrete gains over the
upstream library and makes the fork pair easier to evaluate as a production
tool.
\end{abstract}

\keywords{number-to-words, words-to-number, multilingual parsing, speech,
text normalization}

\section{Introduction}
The original `num2words` library is a mature Python package for rendering
numeric values as words in many languages~\cite{num2words_upstream}. It is
useful, but it is only half of the pipeline. Production systems often need the
inverse operation too: parsing spoken numbers from transcripts, normalizing
hybrid text from LLMs, or round-tripping a number through a human-readable
format and back into a machine value.

The fork pair presented here addresses that gap. `num2words2` preserves the
forward API and extends the conversion surface. `words2num2` consumes the same
locale model and converts words back into numbers. Together they give a single
locale-aware abstraction for both generation and recovery.

\section{What num2words2 Adds}
`num2words2` is not just a repackaging of upstream. The documented surface is
larger and more explicit.

\medskip
\noindent\textbf{Summary.} This matrix covers 143 locale rows. Upstream \texttt{num2words} provides the forward baseline, \texttt{num2words2} expands and documents that surface, and \texttt{words2num2} mirrors the same locale model for reverse parsing, sentence rewriting, and auto-parse workflows.

\subsection{Conversion modes}
Beyond cardinal and ordinal output, `num2words2` documents currency, year,
fraction, and cheque modes. It also accepts string input, so sentence-level
workflows do not need to pre-normalize every token.

\subsection{Aviation and radio phraseology}
The aviation stack is a real differentiator. The \texttt{en\_Aero\_*} family supports
digit-by-digit reading for ICAO, FAA, US Navy, US Army, and NATO profiles,
with explicit phraseology rules for altitude, flight level, heading, squawk,
runway, and frequency handling~\cite{num2words2_reference}. That is more
specific than a generic locale mapping; it is a domain-oriented rendering mode.

\subsection{Bug fixes and quality-of-life changes}
Compared with upstream, the fork is explicit about the fixes it carries:
\begin{itemize}
  \item string input preservation for exact `Decimal`-style values
  \item better handling of negative numbers and floating-point edge cases
  \item more currency coverage, including three-decimal currencies
  \item cheque-style output for bank and accounting use cases
  \item support for fractions as first-class input
  \item per-call options such as \texttt{style=}, \texttt{precision=}, \texttt{cents=},
        \texttt{gender=}, \texttt{case=}, \texttt{spaced=}, and \texttt{decimal\_word=}
\end{itemize}

\section{What words2num2 Adds}
`\texttt{words2num2}` is the inverse library. It parses spoken numbers and can also walk
complete sentences. Its implementation is designed for two different cases:
clean grammar-based parsing for English, and reverse lookup for every other
locale that \texttt{num2words2} exposes~\cite{words2num2_readme}.

\subsection{Reverse conversion}
The core call turns phrases like \texttt{forty-two}, \texttt{nineteen ninety nine}, or
\texttt{twenty-first} into numeric values. That makes the library directly useful for
capturing speech, cleaning form fields, and normalizing generated text.

\subsection{Sentence rewriting}
\texttt{words2num\_sentence} and \texttt{auto\_parse\_sentence} go further. They rewrite mixed
text in place, handling numbers inside sentences instead of only isolated
tokens. That matters for ASR transcripts and LLM output, where digits and words
are usually mixed together.

\subsection{Auto-parse mode}
The auto-parse layer recognizes currencies, units, and locale-specific number
formats. It handles symbols like dollar, euro, pound, yen, rupee, won, and lira, plus
scale shortcuts like \texttt{\$5m}, and it normalizes separators according to locale
defaults and heuristics. The docs describe this as a structured quantity parser,
not just a text replacement helper.

\section{Why This Matters}
The practical gain is round-trippable text normalization.

\paragraph{For speech}
Transcripts often contain long-form numbers, currency values, and units that
need to become machine-readable. `words2num2` parses those forms directly, and
`num2words2` can generate the spoken output again when needed.

\paragraph{For AI and LLM workflows}
Generated text often mixes words, digits, currencies, and units. Sentence-level
rewriting and auto-parse mode make the normalization step local to the library
instead of being spread across ad hoc regexes in application code.

\paragraph{For locale-sensitive applications}
The locale coverage is not an afterthought. Regional variants, aliases, and
aviation-specific profiles are called out explicitly, which makes it easier to
use the same locale identifier across generation and parsing.

\paragraph{For correctness}
The fork pair does not only add features. It also removes pain points that are
common in production: exact decimal handling, negative-number behavior, scale
words, and handling of currencies and fractions that upstream either does not
cover or does not document as clearly.

\section{Discussion}
The main engineering tradeoff is that the fork pair intentionally specializes
the original library. Upstream `num2words` remains a strong baseline for
forward rendering. `num2words2` improves the same domain with stronger
documentation, more explicit locales, and more conversion modes. `words2num2`
adds the missing reverse direction and makes the system useful for real-world
normalization pipelines.

That is the core message: the fork pair is attractive because it is not just a
reimplementation. It is a complete text-normalization stack that covers both
human-readable generation and structured recovery.

\section{Conclusion}
`num2words2` and `words2num2` convert the upstream library into a practical
bidirectional system. The gains are concrete: more explicit locale support,
aviation and radio profiles, sentence and auto-parse handling, stronger
currency and fraction support, and a reverse parser that closes the loop.
Those are the features that make the stack credible for production AI, speech,
and multilingual data-processing workflows.
